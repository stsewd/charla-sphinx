Feature que no está documentado no existe

Antes de hablar sobre sphinx, hablemos de la importancia de la documentación.

Importancia de escribir documentación

  • Transmitir conocimiento de manera rápida y fácil
  • Aprender cómo instalar, usar y configurar el software
  • Aprender qué opciones provee el software
  • Aprender qué interfaces provee el software
  • Aprender cómo luce el software antes de instalarlo
  • Aprender cómo no usar el software

No siempre vamos a estar en un proyecto para responder las preguntas o explicar el cómo de varias cosas.

Feature que no está documentado no existe, el usuario no tiene una manera fácil de saber que ese feature existe.

En caso que sea una API, documentar su interfaz es importante.

Es más fácil atraer la atención de usuarios si ven como luce y las funcionalidades de tu software.

Puede que existan varias formas de realizar una tarea, pero no todas son correctas o las más adecuadas.

Importancia de documentación en proyectos open source

  • Más usuarios usarán tu proyecto
  • Tendrás colaboradores
  • Menos preguntas que responder
  • Y una comunidad más grande

Documentación es muy muy importante en proyectos open source. Proyectos más populares, son populares por si amplia documentación. Como Django, por ejemplo.

Sphinx

https://www.sphinx-doc.org/

pip install sphinx

  • reStructuredText
  • ¡Cross-references!
  • Documentación en prosa y docstrings
  • Extensible
  • HTML, PDF, ePub
  • Temas
  • Multi-lenguaje
  • Python, requests, Flask, Django, Linux kernel

Licencia BSD, licencia open source.

También se puede usar con markdown, veremos las diferencias luego.

Ya no más links hardcodeados en tu documentación, o links que pueden cambiar y producir 404.

Incluye documentación desde tu código, pero también agrega más párrafos desde documentos externos.

Varios documentos para lectura off-line (o para leer en e-readers como kindle).

cd docs
make html
